# TKT-0008 — Two new UI text-style voices: UI-control + UI-widget, with geometry's control text rerouted to UI-control

## Summary
Requested as a text-Style category "UI" for interactive text — two groups: **UI-control** (buttons,
form elements: input, select, textarea, …) and **UI-widget** (compact widgets: tags, badges, switch,
radio, check, …). Clarified at intake (three rulings from the reporter):

1. **Shape — two full voices.** UI-control and UI-widget join the voice system
   (`src/engine/type.mjs` `makeVoices`/`SIZES`) as first-class voices with their own character
   params (weight/tracking/leading/case), per-treatment shaping, and sibling weights — 13 voices
   become 15. Not a size-ladder-only side category.
2. **Steps — 3, like every other voice.** SM/MD/LG only; the reported 6-variant magnitude tables'
   2XL/XL/XS rows are dropped from the voice ramp (recorded below as deferred reference — see
   Scope/Open).
3. **Geometry seam — reroute.** Geometry's per-step composed text size (`-font`,
   `src/engine/geometry.mjs` `buildSize` `fontOverride`, fed by `src/ui/model.mjs` `geometryScale`)
   currently sources the **Label** voice at the matching step (SM/MD/LG only; XS/XL/2XL fall back
   to the `3.16·h^0.45` power law). It reroutes to **UI-control** at the matching steps; the
   XS/XL/2XL power-law fallback stays. (History note: geometry's comment "renamed from 'UI'
   2026-07-13" refers to the OLD voice that became Label — this ticket's "UI" voices are new,
   not a revival of that identity.)

**Authoritative magnitude tables** (reporter-supplied; controls table REVISED mid-intake —
supersedes the seed's original control rows). Columns: Desktop XL | Desktop LG | Desktop | Tablet |

With ruling 2 (SM/MD/LG survive), the voice ramps are:

- **UI-control** `SIZES` = [13, 15, 16] (SM/MD/LG, Desktop); Tablet/Mobile frozen at Desktop

- **UI-widget** `SIZES` = [10, 11, 12]; Tablet/Mobile frozen; Desktop Lg [12, 13, 14];
  Desktop Xl [14, 15, 16].

Reference anchor from the seed: Body LG runs 24/20/18/18/18 across the same columns (matches the
shipped #308 tiers), and UI-control was framed as sitting a fixed offset below Body at every
breakpoint.

## Intake reconciliation (2026-07-16, AskUser — post-intake divergence resolved)

The live session that ran in parallel with this intake shipped a DIFFERENT lightweight
interpretation of the same tables (uncommitted at reconciliation time): a geometry-owned 6-step
`CONTROL_FONT` ramp (the control table verbatim), the type→geometry Label composition RETIRED,
`size/*/font` rows dropped from the Geometry Figma collection, control-text sizes surfaced in the
Typography collection as `UI-widget|UI-control/{SM,MD,LG}/size` variables named by BAND (widget =
control-table XS/SM/MD = 12/13/15 · control = LG/XL/2XL = 16/18/20), and six hand-made text styles
in the BZZR Tokens file bound to those variables + Geometry heights. Three rulings by the reporter
resolve the divergence — **this ticket's model wins on every fork**:

1. **Full voices confirmed** (not the lightweight band shape): UI-control + UI-widget join
   `makeVoices`/`SIZES`; 13 → 15 everywhere.
2. **The widget table is adopted**: UI-widget = its own smaller ramp ([10, 11, 12] Desktop SM/MD/LG
   + its tier columns) — the live build's band semantics for `UI-widget/*` (12/13/15) are
   SUPERSEDED, as are the band values under `UI-control/*` (16/18/20 → the voice's 13/15/16). The
   Figma variable NAMES survive unchanged; only values/semantics move.
3. **Style planner emits the UI styles** — which under ruling 1 falls out of voice citizenship
   (voices flow into the style plan like every other voice); BZZR's six hand-made styles get
   reconciled (superseded by the planner-named set or re-bound) rather than kept as a parallel set.

Execution amendments agreed in the same pass:
- **Geometry XS/XL/2XL fallback = the ratified `CONTROL_FONT` rows** (12/18/20 Desktop ·
  scaled/tier columns), NOT the raw `3.16·h^0.45` power law the intake deferred to — the 6-step
  control table shipped after intake and is fresher intent; the deferred-rows non-goal below is
  amended accordingly. SM/MD/LG compose from the UI-control voice (ruling 3 of the intake), whose
  Desktop values equal those `CONTROL_FONT` cells by construction — the composition is value-neutral

  control text lives with typography).
