## Rubric — an export-format change

Scores adding or changing an output format in ultimate-tokens. `[gate]` = mechanically checkable (`grep` / a named verifier / `npm test`); `[review]` = judgment with cited evidence. Score each 1–5.

| # | Dimension | Type | What it checks | 1 (fail) → 3 (adequate) → 5 (excellent) |
|---|---|---|---|---|
| R1 | Shape validity | [gate] | `node test/engine/exports.mjs` (`leaf-valid`, `padding`, `css-resolves`, `nonempty`) for color; `test/engine/type.mjs` / `geometry.mjs` for type/geom — every emitted token is well-formed (srgb leaf, hex reconstructs from components, correct `$type`, 3-digit padding) | 1: a verifier red (malformed leaf / unpadded stop / format empty) · 3: passes, generic asserts only · 5: passes + a dedicated `[gate]` group added for the new format's load-bearing structure |
| R2 | Reuse-not-reinvent | [review] | The serializer opens `const palettes = derivedAll(state)` (color) and loops `p.stops`/`p.scrims`/`p.roles`; colors print via the shared helpers (`hexOf`/`oklchStr`/`roleOklch`/`colorLeaf`); ref var-names via `refKey`; no hand-rolled color math, no re-walk of `state.palettes` | 1: bespoke hex/oklch formatting or a manual palette walk (disabled-palette / padding bugs reintroduced) · 3: uses `derivedAll`, one ad-hoc format helper · 5: derivation + every color via the shared helpers + every ref var-name via `refKey`, zero new color math |
| R3 | Three-site wiring | [gate] | The serializer is reachable: color → `exportAll` + `model.mjs` `projectView` `exports[id]` (~line 433) + `FORMAT_GROUPS` + `downloadAllZip`; type/geom → `SYSTEM_CODE`/`SYSTEM_LABEL` + the `sys.*` branch of `downloadAllZip`. `npm test` (headless-boot drawer/download) green | 1: serializer exists but absent from a wiring site (blank tab / missing zip entry / not in bundle) · 3: tab + bundle wired, zip entry present · 5: all sites wired, the JSON-vs-string boundary handled, headless-boot asserts it |
| R4 | Two-layer fidelity | [review] | Raw primitives emitted FLAT/mode-independent; the light/dark flip expressed only in the semantic layer via the target's idiom (`light-dark()` / two DTCG mode files / per-mode UI3 aliases / a `{light,dark}` JSON pair). Scrims keyed `500-{step}` (alpha% = step/10) | 1: a mode-branched raw stop, or a single resolved color where a mode pair belongs · 3: correct flip, idiom slightly off · 5: flip in the target's native idiom, raw stays flat, scrims keyed `500-{step}` |
| R5 | ShadCN-by-design | [review] | A new ROLE is NOT auto-spilled into ShadCN; if it should drive a slot, exactly one `SHADCN_ORDER` entry + one `MAP` line added with `:root`/`.dark` parity. Otherwise ShadCN left untouched on purpose | 1: all roles dumped into ShadCN (contract broken) or a mapped token missing from one block · 3: left untouched (correct default) · 5: deliberate single-slot wiring with parity, or a reasoned "left out" |
| R6 | ADR fences respected | [review] | The fenced decisions honored: ADR-002 (default RESOLVED, `rawColl` opt-in for aliasData), ADR-005 (two-layer flat raw), ADR-006 (pad3), ADR-007 (UI3 interchange-only); `geomTokensFigma` stays unitless `$type:"number"` wrapped in `{ Geometry: {…} }` | 1: a fenced decision "fixed" (e.g. default aliasData on, an unpadded key, a px-suffixed Figma number) · 3: fences respected · 5: respected + a new cross-collection need follows the `rawColl` pattern, not a new scheme |
| R7 | Shape documented | [review] | `docs/reference/references/knowledge-04-export-formats.md` gains/updates the format's section; the eight-formats header count kept consistent if a color format was added | 1: shape undocumented (knowledge-04 now stale) · 3: section added · 5: section added + header count + the ShadCN/Figma constraint note kept consistent |

**Gate to ship:** R1 and R3 must each score ≥ 3 — a format whose emitted SHAPE is malformed (R1) or that is not reachable from the UI's wiring sites (R3) is not done, however clean the serializer reads. R6 ≥ 3 is required if the change touches DTCG/Figma/UI3 (a broken ADR fence breaks Figma import).

**Top failure to look for first:** a serializer that exists but is **unwired** (R3) — present in `exports.js` but missing from `exportAll`, `model.mjs` `view.exports`, `FORMAT_GROUPS`, or `downloadAllZip`, so the tab renders blank or the zip lacks the file while every pure verifier still passes. Walk the chain `exportAll → view.exports[id] → FORMAT_GROUPS → downloadAllZip` (or `SYSTEM_CODE → sys.* zip branch` for type/geom) and confirm the format is reachable before trusting a green `node test/engine/exports.mjs`.
