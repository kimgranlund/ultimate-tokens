# TKT-0007 — Corsa (Color Categories → Brands) doesn't reflect authentic Grand-Tour cycling colors

## Summary
Reported as a full 15-role color spec for the "Corsa" brand preset — Neutral, Primary/-muted,
Secondary/-muted, Tertiary/-muted, Quartery/-muted, and Accent-1 through Accent-6 — naming specific
real-world Grand Tour cycling references (Giro d'Italia pink, Vuelta a España red, Tour de France
yellow, team jersey colors, terrain/heritage neutrals). The CURRENT Corsa entry
(`docs/reference/colors/categories/brands.json`, `volumes[0].palettes[4]`, kicker "Corsa · Cycling
club culture · Italy, 1920s–1980s") is a narrower 6-swatch story (Corsa pink, Celeste, Postal navy,
Vintage cream, Racing red, Mustard gold) grouped into only 3 hierarchy buckets (dominant/supporting/
accent) — it does not carry anything resembling the 15 named roles in the report.

## Acceptance
- Corsa's story (swatches + hierarchy, and title/source/refuses if the richer narrative warrants
  it) authentically reflects the 15 role assignments given in the report, each traceable to a real,
  sourced reference (an official Grand Tour jersey color, a documented team kit, or a named
  heritage/terrain reference), not an invented approximation.
- Every color is in-gamut OKLCH, perceptually disciplined, and follows this repo's existing
  category-JSON story schema (see `[[open-schema-question]]` below — the exact target shape is part
  of this ticket's investigation, not assumed).
- `npm test` (`engine/categories.mjs` and any story/schema validators) stays green after the change.

## Links
- `docs/reference/colors/categories/brands.json` — the file to change (Corsa entry).
- `palette-researcher` agent — owns exactly this kind of work (research a theme from authoritative
  sources, emit a curated palette/story in this repo's exact schema).

   Racing red, Mustard gold) don't carry most of these roles at all, and the ones that loosely

## Expected vs actual
- Expected: 15 distinct, named, sourced colors matching the report.
- Actual: 6 swatches in 3 hierarchy buckets (d/s/a), a materially smaller and differently-sourced set.

## Classification
Visual/content — a curated Color Categories preset's color story doesn't match the intended
real-world reference. Two things this investigation must resolve, named here so they aren't silently
guessed:
1. **Open schema question.** Every curated Category preset examined (Corsa itself, its Brands
   siblings Nike/Burger King, and a sampled Architecture preset) uses the same `swatches` (a flat
   list) + `hierarchy` (exactly 3 buckets: dominant/supporting/accent) story shape — never a richer,
   explicitly-named `palettes` array with roles like Neutral/Primary/Secondary/Tertiary/Accent-N.
   That richer shape exists ONLY on the 3 "product's own design system" entries (Maison, Adia,
   BZZR — literal exports, not narrative research). The report's 15-role vocabulary
   (Primary/Secondary/Tertiary/**Quartery**/Accent-1..6) doesn't match either shape exactly — in
   particular "Quartery" and 6 distinct accents aren't a role set seen anywhere else in this
   codebase. Resolve whether this means: (a) express 15 real swatches inside the EXISTING
   swatches/hierarchy shape (more swatches, more hierarchy nuance in the `note`/`text` fields, still
   3 buckets), or (b) this preset genuinely needs the richer explicit-roles shape migrated in — a
   bigger, cross-cutting change since no purely-researched (non-"own product") preset has ever used
   it.
2. **Image gap.** The report illustrated Secondary, Secondary-muted, Tertiary, Tertiary-muted, and
   Quartery with pasted images; those images are not available in this ticket's capture (this ticket
   was minted from a re-invocation of the report, not the original attachment). The named real-world
   references (official Vuelta a España red, Tour de France yellow) are well-documented enough to
   source directly rather than blocking on re-requesting the images — but the dispatched
   investigation should flag it back to Kim if the sourced color doesn't obviously match what a
   "muted lagoon blue" / "limestone/cream" / "newspaper yellow" swatch would look like.

## Severity
minor — Corsa is one of 336+ curated presets in a gallery, not a core engine path; nothing is broken,
the CONTENT is just not what was intended.

## Findings

### 2026-07-13 — palette-researcher investigation (draft written; not applied to brands.json)

**Draft location:** `docs/reference/colors/categories/_drafts/corsa-grand-tour-v2.json`. It holds the
proposed full replacement for `brands.json` `volumes[0].palettes[4]` under a `palette` key, plus two
`_notes_*` blocks. It is NOT auto-consumed — `gen-categories.mjs` only reads `*.json` at the
`categories/` root, not `_drafts/`. To apply: paste the `palette` object over the current Corsa object
in `brands.json`, then run `npm run gen:categories` + `npm test` (both outside this investigation's
tool access). The draft keeps the existing Corsa `type` block (Bodoni Moda / Anton / Söhne / Courier
Prime) verbatim.

**Resolution of the open schema question (option a, with a hard caveat).** Stayed inside the existing
`swatches` + `hierarchy` story shape (per the parent instruction not to imitate the own-product
`palettes` shape). BUT the shape has a mechanical limit that is the real crux here, and it must not be
missed:

- `gen-categories.mjs` `mapColors()` reads **exactly 6 swatches** regardless of how many are authored:
  `dom` = the first `hier:"d"`; `secondary-muted` = the `s` nearest `dom` in OKLab; `tertiary` /
  `tertiary-muted` = the two highest-chroma **remaining** `s`; `primary` / `primary-muted` = the first
  two `a` in array order. **Every other swatch is silently dropped at generation.** So the 15-swatch
  story documents the full 15-role intent legibly (each swatch `name` carries its role, e.g. "Giro
  d'Italia rosa (Primary)"; each `note` says whether it is sourced-fact or a judgment call and whether
  it renders or is dropped), but **only 6 of the 15 actually seed rendered palette families.**
- I ordered/tagged the swatches so the 6 survivors are a clean, iconic Grand-Tour hexad — three proper
  base/muted pairs: **primary = Giro rosa** (+ newspaper-pink muted), **secondary = road blue**
  (+ muted lagoon muted), **tertiary = Tour yellow** (+ yellow-newsprint muted) — plus the derived
  neutral and the four appended status colors. Requires ≥1 `d`, exactly 3 `s`, ≥2 `a` (fewer than 3
  `s` would crash `tertiary-muted`); the draft uses 1 `d` + 3 `s` + 11 `a`.
- **Dropped-at-generation (documented in JSON, not rendered):** Vuelta red, Limestone/cream, Cobbles
  grey, Blue jersey (azzurra), Green jersey (vert), Purple jersey (ciclamino), Gravel brown, Redwood
  brown, Redwood green. Note this drops Vuelta red — a sourced hero color — because only three
  base/muted pairs can render and pink/blue/yellow were the only three that form clean same-hue pairs
  (there is no muted-red partner in the set). If Kim would rather render pink/red/yellow than
  pink/blue/yellow, that is a one-line re-tag but still can't render more than 3 pairs.

**Why 3 buckets genuinely can't do this content full justice (flagging per parent instruction).** The
Grand-Tour story is fundamentally multi-hue — four+ co-equal saturated hues that each MEAN a different
thing (rosa=Giro, jaune=Tour, rojo=Vuelta, plus azzurra/vert/ciclamino classifications). The
swatches/hierarchy shape's 2-2-2 semantics want one ground + working surfaces + one punctuation
base/muted pair — the opposite of "one loud color per tour." Two extra mismatches: (1) the rendered
**Neutral is derived**, not authored, so "Cobbles grey" as Neutral is documentable but not settable;
(2) there is **no "Quartery" and no Accent-1..6** in the app's family model (Neutral + Primary/
Secondary/Tertiary + Info/Success/Warning/Danger = 8 families), so Quartery (Tour yellow) had to be
folded onto `tertiary` and the six named accents can't be distinct families here.

**Recommendation for the full 15 (Kim's call, deliberately not done here).** The faithful vehicle for
a true 15-color render is the `direct palettes` pass-through shape already used by the **"Modal jazz"**
preset in this same file (an authored `palettes` array with `keyColors`/`colorName`/`colorRole`, a
real authored Neutral, and custom status colors — worth noting because the ticket's premise that the
richer shape exists ONLY on own-product entries isn't strictly true; Modal jazz uses it and reads as a
mood, though the file's `sourcing` counts it among the four "own products"). Migrating Corsa to that
shape, or a small `mapColors` change, would let all 15 render — both are bigger than this ticket, so
I've flagged rather than done them.

**Colors — sourced-fact vs. judgment call:**
- *Sourced facts (documented real-world anchors):* Giro **maglia rosa** (pink from La Gazzetta dello
  Sport's pink paper); Tour **maillot jaune** ≈ **Pantone 123 C** (yellow from L'Auto's yellow paper);
  Vuelta **maillot rojo** (leader red since 2010); Giro **maglia azzurra** (mountains, blue since 2012,
  Banca Mediolanum); Giro **maglia ciclamino** (points, cyclamen purple); Tour **maillot vert**
  (points, green). For all of these the CLASSIFICATION/COLOR is sourced; the exact hex/OKLCH is a
  curated "loud graphic poster" read (no org publishes an official Pantone I could confirm for most).
- *Judgment calls (name only, no documented anchor — Kim originally gave several via images I could not
  see):* Secondary "road blue", Secondary-muted "muted lagoon blue", Primary-muted "newspaper pink"
  (grounded in the pink-newsprint origin), Quartery-muted "yellow newspaper" (grounded in L'Auto's
  yellow newsprint), Tertiary-muted "limestone/cream", Neutral "cobbles grey", Accent-4 "gravel brown",
  Accent-5 "redwood brown", Accent-6 "redwood green".

**Things Kim should sanity-check:**
- The five colors he originally supplied via pasted images (Secondary, Secondary-muted, Tertiary,
  Tertiary-muted, Quartery) were reconstructed from the NAMES alone — I could not see the images. In
  particular "muted lagoon blue", "limestone/cream", and "yellow newspaper" are best-guess reads;
  compare them to the pasted swatches.
- "Redwood green" is an unusual name — I read it as a deep pine/forest green. Confirm the intent.
- The hex↔OKLCH pairs are hand-derived estimates (no converter available in this tool context; no
  Bash). They sit in the exemplar chroma register (≤0.20) and were calibrated against the file's
  existing pairs, but running `gen:categories` is the real check — the generator recovers chroma/lift
  from the HEX and hue from the OKLCH string, so a mismatched pair would visibly drift. Worth an
  eyeball after generation.
- The rendered-vs-dropped trade-off (pink/blue/yellow render; Vuelta red + all classification jerseys
  are documented but dropped) is a direct consequence of the 6-swatch cap. If that loss isn't
  acceptable, the `direct palettes` migration above is the path.

### 2026-07-13 — direct-palettes migration authored (Kim's call; draft written, not applied)

Kim reviewed the first pass and chose the `direct palettes` shape (the one "Modal jazz" uses in
`brands.json`) so all of the reported roles render instead of 9 of 15 silently dropping. This entry
records the verified schema ceiling and the authored v3 draft.

**The real ceiling — there is NONE at the family level (verified in code, not assumed).**
- `gen-categories.mjs` sets `direct = Array.isArray(p.palettes) && p.palettes.length ? p.palettes : null`
  (~L229) and emits `palettes: direct || (mapColors…)` (~L255): the authored `palettes` array is passed
  through **verbatim** — no swatch→`mapColors` derivation, no `role-table.json` validation, no
  canonical-name check, **no count cap**, and (unlike the swatches path) **no derived neutral and no
  appended shared status four**. So the direct array must AUTHOR its own `neutral` + `info/success/
  warning/danger` (Modal jazz and BZZR both do).
- `projectView` (`model.mjs` ~L450-514) loops **every** entry of `doc.palettes` and builds that entry's
  own 53 roles from `slug(p.name)`; `persist.clampPalette` keeps `name` as a **free string**. Nothing
  downstream (exports included) assumes a fixed family set. `role-table.json` `defaults[]` (8 families)
  is only a **blank document's starting set**, not a maximum; `rolesPerPalette:53` is roles-per-family.
- **`colorRole` is not a folding mechanism.** `persist.js` (~L155) keeps it only as
  `dominant|supporting|accent`; `app.js` (~L1188-1201) uses it solely to proportion the CONCEPT-STRIP
  preview widths + print a Story-panel label. A family renders fully with or without it.
- Precedent: **Modal jazz already ships 11 families** with non-canonical names (`accent`,
  `accent-muted`, `primary-muted`, `secondary-muted`) and renders fine — proof the ceiling is well
  above 15.
- The only "6" in the path is cosmetic: `app.js` ~L1182 `enabled.slice(0,6)` caps the small concept
  STRIP at 6 bands. All families still render as full 53-role palettes in the canvas + every export.
- **Verdict: all 15 named roles render as DISTINCT families. Nothing folded.**

**v3 draft location:** `docs/reference/colors/categories/_drafts/corsa-grand-tour-v3-direct-palettes.json`.
Same structure as v2 (a `_draft` note + the object under `palette`), plus `_notes_ceiling` +
`_notes_render`. It matches Modal jazz/BZZR's exact field shape and **drops the old top-level
`swatches` and the swatches-only story**, keeping `hierarchy` (d/s/a, for the Story-panel proportions)
+ `dominantHex` + the existing `type` block (verbatim) + a top-level `palettes` array. To apply:
replace the current Corsa object in `brands.json` (`volumes[0].palettes[4]`) with the draft's `palette`
object, then run `npm run gen:categories` + `npm test` (both outside this investigation's tool access).

**Which of the 15 render as distinct families (all of them):** `neutral` (Cobbles grey) · `primary`
(Giro rosa) · `primary-muted` (newspaper pink) · `secondary` (road blue) · `secondary-muted` (muted
lagoon) · `tertiary` (Vuelta red) · `tertiary-muted` (limestone cream) · `quartery` (Tour yellow) ·
`quartery-muted` (yellow newsprint) · `accent-1` (azzurra) · `accent-2` (vert) · `accent-3` (ciclamino)
· `accent-4` (gravel brown) · `accent-5` (redwood brown) · `accent-6` (redwood green). Plus authored
`info/success/warning/danger` (the canonical shared status set, carried explicitly since the direct
path does not append it). **Nothing folded onto `-muted` or `keyColors`** — the v2 drop-list (Vuelta
red, limestone, cobbles grey, azzurra, vert, ciclamino, gravel/redwood browns, redwood green) all now
render as first-class families.

**Sourced-fact vs. judgment-call** is unchanged from the first entry — same colors, now each on its
own family. Sourced classification/origin: maglia rosa (Gazzetta pink paper), maillot jaune (≈Pantone
123 C / L'Auto yellow paper), maillot rojo (Vuelta leader red since 2010), azzurra (Giro mountains,
blue since 2012), maillot vert (Tour points green), ciclamino (Giro points cyclamen). Judgment calls
(name only): road blue, muted lagoon, newspaper pink, yellow newsprint, limestone/cream, cobbles grey,
gravel brown, redwood brown, redwood green.

**What Kim should sanity-check before applying:**
- **Params are estimates; `gen:categories` is the real check.** Each family's `keyColors[0].oklch` is
  the exact sourced color (lossless, source of truth). `hue` = the OKLCH H exactly. `lift` ≈
  `round((OKLCH_L − 0.53) × 115)` — a fit calibrated against Modal jazz's authored (L→lift) pairs.
  `chroma` (%-of-peak, CAM16-based) is estimated by scaling OKLCH C with a hue-band factor read off
  Modal jazz (~330 warm/yellow, ~380 green/magenta, ~470 blue). These shape only the rendered RAMP's
  saturation/placement, not the anchor color — but eyeball the ramps after generation; a mis-estimated
  chroma reads as a slightly hotter/cooler family than intended (no gamut risk — the engine clamps).
- **The five image-sourced colors** (road blue, muted lagoon, Vuelta red as authored, limestone/cream,
  Tour yellow) were still reconstructed from names in the first pass — compare to the original pasted
  swatches if available.
- **"Redwood green"** is read as a deep pine/forest green — confirm the intent.
- **`colorRole` assignment** drives the concept-strip proportions: `primary`=dominant, the grounds/
  neutrals=supporting, the loud jerseys (Vuelta red, Tour yellow, all six accents)=accent. Adjust if a
  different visual-weight read is wanted.
- **Status set** reuses the canonical shared Info/Success/Warning/Danger (same OKLCH Modal jazz uses).
  If Corsa wants a louder, on-theme status set, that's a small authored change.
- Ticket `status` left unchanged per instruction.

miscalibration specific to this preset: it is NOT — the shared `warning` family (hue 73, also
yellow/amber, used identically across all 344 presets in this gallery) shows the exact same muted
characteristic at its own peak-chroma stops. This is a general property of how this engine's tonal-
ramp/damping model renders yellow-band hues, consistent house style across the whole product, not a
defect introduced here — left `quartery`'s params as drafted rather than push it to an
inconsistent-with-everything-else vividness.
