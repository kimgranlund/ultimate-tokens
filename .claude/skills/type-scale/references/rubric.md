## Rubric — a typography-engine change

Scores a change to `src/engine/type.mjs` (+ `scripts/gen-type-fonts.mjs` / `src/ui/type-fonts.js` for a font change). `[gate]` = mechanically checkable (the named verifier group / `npm test`); `[review]` = judgment with cited evidence. Score each 1–5.

| # | Dimension | Type | What it checks | 1 (fail) → 3 (adequate) → 5 (excellent) |
|---|---|---|---|---|
| T1 | The eleven voices + roles | [gate] | every treatment has the ELEVEN voices (Display · Heading · Sub-heading · Kicker · Lead · Body · Quote · Caption · UI · Code · Legal) + a `fonts` palette; `roleOf` maps Kicker + Code → `mono`, Lead → `body`, Quote → `heading`, Caption + Legal → `ui`, Display/Heading/Sub-heading/UI → their roles; the `box` decoupling holds — Caption + Legal ride the `ui` FONT but are PROSE (`box:false`), so they emit NO `singleLineHeight` (the 11-voices + role-mapping asserts in `test/engine/type.mjs`) | 1: a treatment missing a voice / wrong role mapping / a ui-font prose voice emitting a single-line height (group red) · 3: all eleven present, roles correct, box/prose split right · 5: + the `GROUPS` list, the `persist.js` VOICES allowlist, and the docs/reference voice table stay consistent with the code |
| T2 | Modular-scale math | [gate] | `buildCategory` derives every size (modular scale → 8px floor → the nice-number ladder); Body MD = `bodyBase`; sizes sit on the ladder and strictly increase XS→XL; `LG/MD ≈ ratio`; `lineHeight = round(size·leading)`; Code/UI carry the 8-step `STEPS_UI` ramp | 1: a non-monotonic ramp, Body MD ≠ bodyBase, an off-ladder size, or a hand-authored px size in a treatment (gate red) · 3: monotone + ladder + ratio-correct for the default · 5: + `bodyBase` scales the whole system uniformly (Display grows with Body), ratios preserved |
| T3 | Optical tracking + leading | [gate]+[review] | Display tracks negative + scales with size (XL more negative than XS); UI / caps Headings track positive; `trackingEm` is an em coefficient (not flat px); leadings at the FIXED per-role constants (display **0.8** < 1 · heading · sub-heading **1.125** · prose **1.5** · Lead **1.4** · Quote **1.35** · Caption · Legal **1.5** · Kicker **1.4** · code **~1.5** · UI **~1.4**), uniform across treatments | 1: Display tracking positive/flat, Display leading ≥ 1, or a leading off its per-role constant · 3: signs correct, leadings at the constants for defaults · 5: + tracking demonstrably optical across all voices, leadings verified per voice against the intent |
| T4 | Case discipline | [gate] | Sub-heading + Kicker are UPPERCASE caps voices; Display is title/sentence EXCEPT exactly ONE treatment (Brutalist/`statement`) is ALL-CAPS (the exactly-one-uppercase-Display assert); caps Headings track positive, Display caps negative | 1: a second uppercase Display, Display forced caps everywhere, or a caps voice lost its uppercase (gate red) · 3: exactly one uppercase Display, caps voices intact · 5: + tracking signs correct on every caps path |
| T5 | The font-quoting guard | [gate] | `typeTokensCSS` emits `--font-{role}: '{family}'` QUOTED; `typeTokensCSS(typeScale({treatment:"luxury"}))` contains `--font-display: 'Source Serif 4'` (the luxury quoting assert) | 1: an unquoted family name (Safari drops it → fallback; the smoke is Chrome-only so it stays green) · 3: quoted, gate green · 5: quoted + verified the rendered specimen in Safari, not just Chrome |
| T6 | Emitter shapes | [gate] | `typeTokensCSS` = custom props + a utility class per step pointing at `var(--font-{role})`; `typeTokensDTCG` = a `fontFamily` group + composite `typography` tokens (dimension strings + NUMBER weight); the responsive `@media` + Figma-modes emitters honor the same resolved scale | 1: a malformed token / wrong `$type` / weight as a string (gate red) · 3: the emitters green for the default · 5: + emitters operate purely on the resolved `scale` (no re-run math), `textTransform`→`textCase` carried through |
| T7 | Font wiring (only if fonts changed) | [review] | a new/swapped family is in BOTH the treatment's `fonts` AND `scripts/gen-type-fonts.mjs#FAMILIES` (name + `wght` axis); `npm run gen:type-fonts` ran; `src/ui/type-fonts.js` committed | 1: family in `fonts` but not embedded (renders fallback) or `type-fonts.js` hand-edited/stale · 3: both ends wired, regenerated · 5: + axis correct, specimen confirmed in the real face offline/in-plugin |
| T8 | Purity + determinism | [gate] | `type.mjs` stays pure (no DOM/RNG/clock); sizes integer, `letterSpacing` 2-dp; same config → identical tokens; `node test/engine/type.mjs` + `npm test` green | 1: nondeterministic output or a DOM dependency in the engine · 3: pure, deterministic · 5: + the change adds no new rounding/precision drift |

**Gate to ship:** T1, T2, T4, T5, T6, T8 must each score ≥ 3 — `node test/engine/type.mjs` AND `npm test` green, with the eleven voices intact, the modular scale monotone, exactly one uppercase Display, and the font names QUOTED. A change that drops a voice (T1), breaks the ramp (T2), forces/loses an uppercase Display (T4), emits an unquoted family (T5), or malforms a token (T6) is not done regardless of how good the specimen looks.

**Top failure to look for first:** an UNQUOTED font family (T5) — it passes the Chrome-only smoke and looks fine, but Safari drops the declaration and renders the fallback. The `typeTokensCSS(luxury)` quoting assert is the backstop; confirm it (and eyeball Safari) before trusting "looks done." A close second: forcing Display to UPPERCASE across treatments (T4) — the "exactly ONE uppercase Display" filter turns red the moment a second treatment opts in.
