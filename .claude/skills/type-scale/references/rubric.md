## Rubric — a typography-engine change

Scores a change to `src/engine/type.mjs` (+ `scripts/gen-type-fonts.mjs` / `src/ui/type-fonts.js` for a font change). `[gate]` = mechanically checkable (the named verifier group / `npm test`); `[review]` = judgment with cited evidence. Score each 1–5.

| # | Dimension | Type | What it checks | 1 (fail) → 3 (adequate) → 5 (excellent) |
|---|---|---|---|---|
| T1 | The seven voices + roles | [gate] | every treatment has the SEVEN groups (Display · 3 Headings · Body · UI · Code) + a `fonts` palette; `roleOf` maps Eyebrow + Code → `mono`, Display/Heading/Body/UI → their roles (`test/engine/type.mjs:11,17–18`) | 1: a treatment missing a group / wrong role mapping (group red) · 3: all seven present, roles correct · 5: + the `GROUPS7` list (`test:9`) and the docs/spec "seven groups" table stay consistent with the code |
| T2 | Modular-scale math | [gate] | `size = max(8, round(base·factor·ratio^n))`; MD = base·factor; sizes strictly increase XS→XL; `LG/MD ≈ ratio`; `lineHeight = round(size·leading)`; Code/UI carry the 8-step `STEPS_UI` ramp (`test:38–46`) | 1: a non-monotonic ramp, MD ≠ base·factor, or a hand-authored px size in a treatment (gate red) · 3: monotone + ratio-correct for the default · 5: + `bodyBase` scales the whole system uniformly (Display grows with Body, `test:62–65`), ratios preserved |
| T3 | Optical tracking + leading | [gate]+[review] | Display tracks negative + scales with size (XL more negative than XS, `test:52,55`); UI / caps Headings track positive (`test:53,28`); `trackingEm` is an em coefficient (not flat px); leadings inside the bands (display 1.05–1.2 · heading 1.05–1.3 · prose 1.45–1.65 · UI 1.25–1.5 · mono ~1.5) | 1: Display tracking positive/flat, or a leading outside its band · 3: signs correct, bands respected for defaults · 5: + tracking demonstrably optical across all voices, leadings band-checked per voice |
| T4 | Case discipline | [gate] | Context + Eyebrow are UPPERCASE caps voices (`test:21`); Display is title/sentence EXCEPT exactly ONE treatment (Brutalist/`statement`) is ALL-CAPS (`test:25–26`); caps Headings track positive, Display caps negative (`test:28–29`) | 1: a second uppercase Display, Display forced caps everywhere, or a caps voice lost its uppercase (gate red) · 3: exactly one uppercase Display, caps voices intact · 5: + tracking signs correct on every caps path |
| T5 | The font-quoting guard | [gate] | `typeTokensCSS` emits `--font-{role}: '{family}'` QUOTED (`type.mjs:122`); `typeTokensCSS(typeScale({treatment:"luxury"}))` contains `--font-display: 'Source Serif 4'` (`test:78–79`) | 1: an unquoted family name (Safari drops it → fallback; the smoke is Chrome-only so it stays green) · 3: quoted, gate green · 5: quoted + verified the rendered specimen in Safari, not just Chrome |
| T6 | Emitter shapes | [gate] | `typeTokensCSS` = custom props + a utility class per step pointing at `var(--font-{role})` (`test:74–75`); `typeTokensDTCG` = a `fontFamily` group + composite `typography` tokens (px size strings + NUMBER weight, `test:85–87`) | 1: a malformed token / wrong `$type` / weight as a string (gate red) · 3: both emitters green for the default · 5: + emitters operate purely on the resolved `scale` (no re-run math), `textTransform`→`textCase` carried through |
| T7 | Font wiring (only if fonts changed) | [review] | a new/swapped family is in BOTH the treatment's `fonts` AND `scripts/gen-type-fonts.mjs#FAMILIES` (name + `wght` axis); `npm run gen:type-fonts` ran; `src/ui/type-fonts.js` committed | 1: family in `fonts` but not embedded (renders fallback) or `type-fonts.js` hand-edited/stale · 3: both ends wired, regenerated · 5: + axis correct, specimen confirmed in the real face offline/in-plugin |
| T8 | Purity + determinism | [gate] | `type.mjs` stays pure (no DOM/RNG/clock); sizes integer, `letterSpacing` 2-dp; same config → identical tokens; `node test/engine/type.mjs` + `npm test` green | 1: nondeterministic output or a DOM dependency in the engine · 3: pure, deterministic · 5: + the change adds no new rounding/precision drift |

**Gate to ship:** T1, T2, T4, T5, T6, T8 must each score ≥ 3 — `node test/engine/type.mjs` AND `npm test` green, with the seven groups intact, the modular scale monotone, exactly one uppercase Display, and the font names QUOTED. A change that drops a voice (T1), breaks the ramp (T2), forces/loses an uppercase Display (T4), emits an unquoted family (T5), or malforms a token (T6) is not done regardless of how good the specimen looks.

**Top failure to look for first:** an UNQUOTED font family (T5) — it passes the Chrome-only smoke and looks fine, but Safari drops the declaration and renders the fallback. The `typeTokensCSS(luxury)` quoting assert (`test:78–79`) is the backstop; confirm it (and eyeball Safari) before trusting "looks done." A close second: forcing Display to UPPERCASE across treatments (T4) — the "exactly ONE uppercase Display" filter (`test:26`) turns red the moment a second treatment opts in.
