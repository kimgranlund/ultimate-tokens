# TKT-0016 — schemaVersion + explicit rename maps for saved-doc hydration

## Summary
Saved docs from before a canon rename silently lose data on hydrate — persist.js's allowlist model DROPS a voice absent from VOICES (its own comment admits it), and the 2026-07-13 voice renames are live examples. Add schemaVersion to serialized docs now (cheap, non-breaking) and adopt the hueSpace legacy-stamp pattern: explicit old→new voice/field maps at hydrate. (docs/reference/reviews/2026-07-17-cto-app.md finding 5.)

## Acceptance
- serialize() stamps schemaVersion; hydrate() runs versioned rename maps before clamp
- Pre-2026-07-13 doc fixture round-trips with overrides intact
- Convention recorded: every future canon rename adds its map
