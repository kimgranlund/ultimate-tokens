## Rubric — a typography-engine change

Scores a change to `src/engine/type.mjs` (+ `scripts/gen-type-fonts.mjs` / `src/ui/type-fonts.js` for a font change). `[gate]` = mechanically checkable (`node test/engine/type.mjs` / `npm test`); `[review]` = judgment with cited evidence. Score each 1–5.

| # | Dimension | Type | What it checks | 1 (fail) → 3 (adequate) → 5 (excellent) |
|---|---|---|---|---|
| T1 | The fifteen voices + roles | [gate] | every treatment has the FIFTEEN voices (Display · Headline · Sub-heading · Title · Sub-title · Lead · Body · Body-mono · Label · Label-mono · Kicker · Tiny · Tiny-mono · UI-control · UI-widget) + a `fonts` palette; thirteen ride SM/MD/LG and the two interactive voices ride XS..2XL (`ranksFor` by `SIZES` length); `roleOf` maps Body-mono/Label-mono/Kicker/Sub-title/Tiny-mono → `mono`, Display/Headline/Sub-heading/Title → their roles, Body/Lead → `body`, Label/Tiny/UI-control/UI-widget → `ui`; the `box` decoupling holds — the BOX voices are exactly Kicker/UI-control/UI-widget (emit `singleLineHeight`; Label/Body-mono/Label-mono prose since 2026-07-16), every other voice is prose | 1: a treatment missing a voice / wrong role mapping / a prose voice emitting a single-line height (assert red) · 3: all fifteen present, roles correct, box/prose split + per-voice ramps right · 5: + the count literals (`GROUPS`, `persist.js`'s VOICES allowlist, `test/ui/headless-boot.mjs`'s 51/15, the docs/reference voice table) stay consistent with the code |
| T2 | The fixed-size-table math | [gate] | every voice's SIZE is the literal `SIZES[voice][step]` (no modular ratio); `factor = bodyBase/16` scales the WHOLE table uniformly; an UNSCALED size (factor 1, no breakpoint compression) passes through EXACT, never re-snapped to a different "nice" number; a genuinely scaled/compressed size still lands on the nice-number ladder and strictly increases SM→LG; `leadingRatio`/`trackingRatio` stay the EXACT unrounded ratio (never re-derived from the rounded `lineHeight`/`letterSpacing`) | 1: a hand-authored size gets silently re-rounded, a scaled ramp is non-monotonic, Body MD ≠ bodyBase, or a relative-unit emitter re-derives its ratio from a rounded absolute (gate red) · 3: exact passthrough + monotone scaled ramp for the default · 5: + `bodyBase` scales the whole system uniformly, `modeFactor` breakpoint compression verified against the canonical Tablet 5/6 · Mobile 2/3 factors |
| T3 | Optical tracking + leading | [gate]+[review] | Display tracks negative + scales with the derived size (LG more negative than SM); Label/Kicker/caps-Headings track positive; `trackingEm` is an em coefficient (not flat px); leadings at the FIXED per-role constants (display **0.8** < 1, heading-family **1.125**, prose **1.4–1.5**, single-line control text **1.0**), uniform across treatments; `parseRatio` accepts both a unitless ratio and a percent string (`"112.5%"`) | 1: Display tracking positive/flat, Display leading ≥ 1, or a leading off its per-role constant · 3: signs correct, leadings at the constants for defaults · 5: + tracking demonstrably optical across all voices, leadings verified per voice against the intent |
| T4 | Case discipline | [gate] | Sub-heading + Kicker are UPPERCASE caps voices; Display is title/sentence EXCEPT exactly ONE treatment (Brutalist/`statement`) is ALL-CAPS; caps voices track positive, Display caps negative | 1: a second uppercase Display, Display forced caps everywhere, or a caps voice lost its uppercase (gate red) · 3: exactly one uppercase Display, caps voices intact · 5: + tracking signs correct on every caps path |
| T5 | The font-quoting guard | [gate] | `typeTokensCSS` emits BOTH `--font-{role}: '{family}'` and `--font-voice-{voice}: '{family}'` QUOTED; `typeTokensCSS(typeScale({treatment:"luxury"}))` contains `--font-display: 'Source Serif 4'` | 1: an unquoted family name at either level (Safari drops it → fallback; the smoke is Chrome-only so it stays green) · 3: quoted, gate green · 5: quoted + verified the rendered specimen in Safari, not just Chrome |
| T6 | Body-class core-weight snap boundary | [gate] | every body-class voice's (`BODY_CLASS_VOICES`: Lead/Body/Body-mono/Label/Label-mono/Tiny/Tiny-mono) core weight resolves, via `weightNameFor`'s nearest-ladder-stop, to Regular(400) when its style is labeled "regular •" — i.e. the core weight stays ≤450 | 1: a body-class core weight sits above 450 and silently resolves to Medium(500) or heavier while its Figma label still reads "regular •" · 3: every body-class core ≤450 for the engine defaults and any preset touched by the change · 5: + verified against a real preset's actual font (weight-ladders-and-labels.md's preset-authoring rule) |
| T7 | Emitter shapes | [gate] | `typeTokensCSS` = custom props + a utility class per step; `typeTokensDTCG` = `fontFamily` (15 voices) + composite `typography` tokens (relative lineHeight/letterSpacing, px size/paragraphSpacing, numeric weight) + an optional `weights` group; `typeTokensFigmaModes` (the type/ half of the merged breakpoint-moded "Geometry" collection, TKT-0009)/`typeTokensFigmaPrimitives` emit ALL-PIXEL values (the one Figma exception to the relative-unit rule) | 1: a malformed token / wrong `$type` / weight as a string / a Figma export emitting a percent instead of pixels (gate red) · 3: the emitters green for the default · 5: + emitters operate purely on the resolved `scale` (no re-run math) |
| T8 | Font wiring (only if fonts changed) | [review] | a new/swapped family is in BOTH the treatment's `fonts` AND `scripts/gen-type-fonts.mjs#FAMILIES` (name + `wght` axis); `npm run gen:type-fonts` ran; `src/ui/type-fonts.js` committed | 1: family in `fonts` but not embedded (renders fallback) or `type-fonts.js` hand-edited/stale · 3: both ends wired, regenerated · 5: + axis correct, specimen confirmed in the real face offline/in-plugin |
| T9 | Purity + determinism | [gate] | `type.mjs` stays pure (no DOM/RNG/clock); sizes integer, `letterSpacing` 2-dp; same config → identical tokens (incl. auto-populated sibling weights); `node test/engine/type.mjs` + `npm test` green | 1: nondeterministic output or a DOM dependency in the engine · 3: pure, deterministic · 5: + the change adds no new rounding/precision drift |

**Gate to ship:** T1, T2, T4, T5, T6, T7, T9 must each score ≥ 3 — `node test/engine/type.mjs` AND `npm test` green, with the fifteen voices intact, the fixed-size table exact-passthrough when unscaled, exactly one uppercase Display, font names QUOTED at both levels, and body-class cores ≤450. A change that drops a voice (T1), silently re-rounds a hand-authored literal or breaks ratio-constancy (T2), forces/loses an uppercase Display (T4), emits an unquoted family (T5), lets a body-class core drift past the snap boundary (T6), or malforms a token (T7) is not done regardless of how good the specimen looks.

**Top failure to look for first:** an UNQUOTED font family (T5) — it passes the Chrome-only smoke and looks fine, but Safari drops the declaration and renders the fallback. A close second: a body-class core weight silently drifting past 450 (T6) — the style still LABELS as "regular •" but renders Medium, and nothing in a casual review catches it without checking the resolved face. A third: re-deriving a relative CSS/DTCG unit from a rounded absolute pixel instead of the stored `leadingRatio`/`trackingRatio` (T2) — passes a same-step check but drifts across steps.
